**Scenario 1 – Breast Cancer Prediction (KNN Classification)**
In this project, a **K-Nearest Neighbors (KNN) classifier** is used to predict whether a tumor is **benign or malignant** based on medical measurements such as radius, texture, perimeter, area, and smoothness. The dataset used is the Breast Cancer Dataset available on Kaggle. The workflow includes loading and inspecting the dataset, preprocessing the data, encoding the diagnosis labels, applying feature scaling (which is important for KNN), splitting the data into training and testing sets, and training the model. Different values of **K** are tested to analyze model performance, followed by evaluation using accuracy, precision, recall, and F1-score. Visualizations such as a confusion matrix, accuracy vs K plot, and decision boundary are used to better understand the model’s behavior. Dataset link: [https://www.kaggle.com/datasets/yasserh/breast-cancer-dataset](https://www.kaggle.com/datasets/yasserh/breast-cancer-dataset)

**Scenario 2 – Loan Approval Prediction (Decision Tree Classifier)**
This project also includes building a **Decision Tree classifier** to predict whether a loan application should be **approved or rejected** based on factors such as applicant income, loan amount, credit history, education, and property area. The dataset used is the Loan Prediction Dataset from Kaggle. The process involves loading the dataset, handling missing values, encoding categorical variables, splitting the dataset into training and testing sets, and training the decision tree model. The project further explores tree depth and pruning to understand overfitting and compares shallow versus deep trees. Model performance is evaluated using accuracy, precision, recall, and F1-score, along with visualizations such as a confusion matrix, tree structure plot, and feature importance plot. Dataset link: [https://www.kaggle.com/datasets/ninzaami/loan-predication](https://www.kaggle.com/datasets/ninzaami/loan-predication)
